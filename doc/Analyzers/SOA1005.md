# SOA1005

## Implement either IInitializableModule or IConfigurableModule

When Optimizely discovers an initializable module and creates an instance, it
casts the instance to `IInitializableModule` so it can call the
`Initialize(InitializationEngine)` method. If the instance does not implement
that interface the cast will fail, an exception will be thrown, and the site
will fail to start.

This analyzer verifies that any class decorated with
`EPiServer.Framework.Initialization.InitializableModuleAttribute` or
`EPiServer.Framework.Initialization.ModuleDependencyAttribute` also implements
one of the required interfaces: `EPiServer.Framework.IInitializableModule` or
`EPiServer.ServiceLocation.IConfigurableModule`. Optimizely expects classes
annotated with these attributes to implement one of these interfaces;
otherwise the runtime cast described above can fail. If a class has one of the
attributes but does not implement either interface, this analyzer reports an
error.

Implement `IInitializableModule` in most cases. Implement `IConfigurableModule`
only when you need the `ConfigureContainer(ServiceConfigurationContext)` method;
`IConfigurableModule` extends `IInitializableModule`.

## Example

This will trigger SOA1005:
```C#
// namespace and usings omitted

[InitializableModule]
public class MyInitModule // <-- SOA1005
{
	// ...
}

```

Fix:
```C#
// namespace and usings omitted

[InitializableModule]
public class MyInitModule : IInitializableModule
{
	// ...
}
```
