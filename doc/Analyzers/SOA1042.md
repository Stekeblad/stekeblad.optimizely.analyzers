# SOA1042

## The given content type cannot be instantiated

This analyzer reports an error for calls to `IContentRepository.GetDefault<T>()`
with a T that would result in Optimizely throwing a runtime exception.

For the common cases the following criteria must be met for T:

- The type is not abstract.
- The type is not an interface.
- The type is decorated with `ContentTypeAttribute`.

With the following exceptions:

- `ContentFolder` and `ContentAssetFolder` are allowed
even though they are not decorated with `ContentTypeAttribute`.
- If T is a type parameter then it's ignored by the analyzer because 
the actual type is unknown at compile time and of the complexity
and potential performance impact of checking the call sites of the containing method.

## Example

```C#
	using EPiServer;
	using EPiServer.Core;
	using EPiServer.DataAnnotations;

	// Given these types
	public abstract class MyBasePage : PageData { }

	public interface IContentWithHeading : IContentData
	{
		string Heading { get; set; }
	}

	[ContentType]
	public class MyPage : MyBasePage, IContentWithHeading
	{
		public virtual string Heading { get; set; }
	}

	public class MyOtherPage : MyBasePage, IContentWithHeading
	{
		public virtual string Heading { get; set; }
	}

	// The following lines will generate SOA1042
	_contentRepository.GetDefault<MyBasePage>(parentLink); // MyBasePage is abstract
	_contentRepository.GetDefault<MyOtherPage>(parentLink); // MyOtherPage is not decorated with ContentTypeAttribute
	_contentRepository.GetDefault<IContentWithHeading>(parentLink); // IContentWithHeading is an interface

	// But this one will not
	_contentRepository.GetDefault<MyPage>(parentLink);

	public static T CreateBelowStart<T>(this IContentRepository contentRepo) where T : MyBasePage
	{
		// This call will be ignored (detailed above)
		return contentRepo.GetDefault<T>(ContentReference.StartPage);
	}
```

## Analyzer Change Log

Initially released in version 1.10.0. [View the 1.10.0 version](https://github.com/Stekeblad/stekeblad.optimizely.analyzers/blob/v1.10.0/doc/Analyzers/SOA1042.md)

Version 1.10.1 fixed a few edge cases where the analyzer
would report false positives for `ContentFolder` and `ContentAssetFolder`.
The analyzer now also ignores calls to `GetDefault<T>` when T
is a type parameter to avoid false positives inside generic helper methods.
